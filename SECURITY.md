# Security Policy

## Reporting a Vulnerability

Please do not open public issues for suspected security vulnerabilities.

Instead:

- Use GitHub Security Advisories / private vulnerability reporting:
  <https://github.com/hariria/wellformed/security/advisories/new>
- Include:
  - affected component/crate/package
  - impact summary
  - reproduction details
  - suggested mitigation (if available)

Before making the repository public, enable GitHub private vulnerability reporting
or add a monitored security contact here. Do not rely on public issues for
vulnerability intake.

## Response Targets

- Initial acknowledgment: within 3 business days
- Triage decision: within 7 business days
- Fix timeline: depends on severity and release risk

## Disclosure

- We prefer coordinated disclosure.
- Credit is provided unless you request anonymity.
